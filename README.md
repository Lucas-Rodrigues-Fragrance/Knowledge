# Knowledge — Base de connaissances (Obsidian)

Coffre (*vault*) Obsidian synchronisé via Git, dédié à la recherche
interdisciplinaire et à la préparation de candidatures doctorales.

## Objectif

Construire et amplifier une base de connaissances au service d'un projet de
recherche sur **l'influence de la technologie sur les individus**, dans une
perspective interdisciplinaire (philosophie, **STS — Science & Technology
Studies**, sciences de la communication).

**Cible** : candidatures PhD aux États-Unis (Cornell, Stanford, MIT, etc.) et
ailleurs, depuis un parcours EHESS / ENS (master interdisciplinaire en
philosophie).

## Comment ça marche

Ce dépôt **est** un coffre Obsidian. Le contenu est constitué de fichiers
Markdown (`.md`) que l'on peut éditer dans Obsidian sur votre machine, puis
synchroniser automatiquement ici grâce au plugin **Obsidian Git**.

➡️ Voir le guide d'installation : [`docs/SETUP-Obsidian-Git.md`](docs/SETUP-Obsidian-Git.md)

## Structure du coffre

| Dossier | Rôle |
|---|---|
| `00-Inbox/` | Capture rapide, notes brutes à trier |
| `01-Notes/` | Notes permanentes (méthode zettelkasten) — `Concepts/`, `Auteurs/`, `Lectures/` |
| `02-Sources/` | Références bibliographiques : articles, livres, archives |
| `03-Candidatures-PhD/` | Tout le matériel de candidature (SoP, writing sample, universités cibles) |
| `04-MOC/` | *Maps of Content* — index thématiques reliant les notes |
| `templates/` | Modèles de notes réutilisables dans Obsidian |

## Principes

- **Atomicité** : une note = une idée, reliée aux autres par des liens `[[...]]`.
- **Interdisciplinarité assumée** : relier philosophie, STS, communication, et terrains empiriques.
- **Traçabilité** : toute affirmation s'appuie sur une source identifiée dans `02-Sources/`.
